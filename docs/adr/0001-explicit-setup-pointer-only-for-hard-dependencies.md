# 仅在硬依赖时显式引用 `/setup-matt-pocock-skills`

工程类技能依赖于通过 `/setup-matt-pocock-skills` 注入的每仓库配置（issue 跟踪器、triage 标签词汇表、领域文档布局）。有些技能没有这些配置就无法正常运作——它们需要发布到特定的 issue 跟踪器或应用特定的标签字符串。另一些技能只用这些配置来增强输出（词汇表、ADR 感知），没有配置也能优雅降级。

我们将技能分为**硬依赖**和**软依赖**两类：

- **硬依赖**（`to-issues`、`to-prd`、`triage`）——包含一个显式的单行提示：_"……应该已经提供给你了——如果没有，运行 `/setup-matt-pocock-skills`。"_ 没有映射时，输出是错误的，不仅仅是模糊。
- **软依赖**（`diagnose`、`tdd`、`improve-codebase-architecture`、`zoom-out`）——仅在松散的行文中引用"项目的领域词汇表"和"你所触及领域的 ADR"。如果这些文档不存在，技能仍然可以工作；输出只是没那么精准。

这种划分让软依赖技能保持轻量 token 消耗，避免将 setup 引用机械地复制到并非承重的地方。
