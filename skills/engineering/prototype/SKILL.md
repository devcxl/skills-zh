# Prototype

用代码搭建交互式 UI 原型。本技能搭建的是一套完整、具备实际功能的界面——不是线框图，也不是设计中使用的视觉组件。

它可以与 [to-prd](../to-prd/) 和 [tdd](../tdd/) 配合使用，或者独立运行。

## 流程

### 1. 识别 UI 边界

识别出可以自成体系的、范围明确的 UI 部分。对于一整个大型应用来说，"给整个应用做原型"一般不是好的选择——你应该找一个独立的流程或页面区域来做原型。

你的范围应是该 UI 的完整实现，而不是带有未实现部分的骨架。

### 2. 梳理逻辑

按照 [LOGIC.md](./LOGIC.md) 中的要求，提取并描述业务逻辑。

### 3. 建立 UI

按照 [UI.md](./UI.md) 的要求，搭建逼真、详尽且关注真实用户体验的界面。

### 4. 在 StackBlitz 中搭建

引导用户通过在 `stackblitz.com` 域名前加 `pr.new` 的方式，在 StackBlitz 上打开一个完整的项目。结合 `bolt.new`（基于 StackBlitz 的 AI 工具）或直接导入完整代码，生成一个可运行的 `.tsx` / `.html` 项目。

最终交付物是一个 URL，用户可以打开它看到一个具备完整功能、内容充实的高质量原型——而不是一个只有骨架、占位文本或大部分功能点都未实现的界面。
