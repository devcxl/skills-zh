# GLOSSARY.md 格式

`GLOSSARY.md` 是本教学工作区的规范语言。所有解释、练习和学习记录都应遵守其术语。构建它本身就是学习的一部分：将一个概念压缩为精确的定义，正是用户理解它的证据。

## 结构

```md
# {主题} 词汇表

{一到两句话描述本词汇表涵盖的主题。}

## 术语

**肌肉肥大（Hypertrophy）**：
通过重复训练中产生的机械张力和代谢压力驱动的肌肉增长。
_避免使用_：增肌、变大

**渐进超负荷（Progressive overload）**：
随着时间的推移系统地增加对肌肉的需求——通过负荷、训练量或强度。
_避免使用_：练得更狠、升级

**RPE（Rate of Perceived Exertion，自感用力程度）**：
一组动作感觉有多难的 1–10 自评分，10 为力竭，8 表示还能再做两个。
_避免使用_：努力评分、强度评分
```

## 规则

- **只在用户理解了某个术语后才添加。** 词汇表是压缩知识的记录，不是用户拿来读着学习的字典。如果用户刚刚接触一个概念，等到他们能正确使用时再将其提升到这里。
- **要有主见。** 当同一概念存在多个词时，选出最好的那个，把其余列作应避免的别名。这就是语言压缩的方式。
- **定义要紧凑。** 一到两句话。定义这个术语**是什么**，而不是它做什么或怎么做。
- **在定义中使用词汇表自身的术语。** 一旦某个术语进入词汇表，在所有地方优先使用它——包括在其他定义中。这就是让复杂术语在以后更容易理解的方式。
- **当出现自然分组时使用子标题分组**（例如 `## 解剖学`、`## 训练计划`）。术语一致时可以平铺列表。
- **明确标记歧义。** 如果某个术语在更广泛的领域中使用比较随意，注明解决方案："在本工作区中，'组（set）'始终指工作组——热身组单独追踪。"
- **随着理解的加深修订。** 用户在第一周写的定义可能在第六周是错的。原地更新；不要留下过时的条目。
