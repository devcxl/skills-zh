# GLOSSARY.md 格式

`GLOSSARY.md` 是本教学工作区的规范语言。所有讲解、练习和学习记录都应遵循其术语。构建它本身就是学习的一部分：将一个概念压缩成精确定义是用户理解它的证据。

## 结构

```md
# {主题} 术语表

{一两句话描述本术语表涵盖的主题。}

## 术语

**Hypertrophy（肌肉肥大）**：
由机械张力和代谢压力在多次训练中驱动的肌肉生长。
_避免使用_：增肌、变大

**Progressive overload（渐进超负荷）**：
随时间系统性地增加对肌肉的需求——通过负重、容量或强度。
_避免使用_：更努力、升级

**RPE（主观努力程度评分）**：
1-10 分的自我评分，表示一组动作的吃力程度，10 为力竭，8 表示还剩两次重复。
_避免使用_：努力分数、强度评分
```

## 规则

- **仅在用户理解该术语时才添加。** 术语表是压缩知识的记录，而非用户阅读学习的字典。如果用户刚接触一个概念，等到他们能正确使用它之后再将其收录于此。
- **要有观点。** 当同一个概念有多个词语时，选择最好的一个，并将其他列为应避免的别名。这就是语言压缩的方式。
- **保持定义紧凑。** 一两句话。定义术语*是什么*，而非它做什么或如何做。
- **在定义内部使用术语表自己的术语。** 一旦某个术语进入术语表，优先在任何地方使用它——包括在其他定义内部。这就是使复杂术语后来更容易理解的方式。
- **当自然聚类出现时按子标题分组**（例如 `## 解剖学`、`## 编程`）。当术语之间紧密相关时，扁平列表也可以。
- **明确标记歧义。** 如果某个术语在更广泛的领域中被松散使用，注明解决方案："在本工作区中，'组'始终指工作组——热身组单独记录。"
- **随理解加深而修订。** 用户第一周写下的定义可能在第六周是错误的。就地更新；不要留下过时的条目。
